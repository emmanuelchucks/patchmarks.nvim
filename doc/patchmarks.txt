1. Introduction                                             *patchmarks-intro*

Patchmarks is a file-oriented Git review plugin.

It starts a review session for the current Git change set and lets you attach
freeform annotations to lines or line ranges in normal file buffers. Those
annotations can then be exported back to an LLM coding agent.

Patchmarks does not render diffs itself.

==============================================================================
2. Setup                                                     *patchmarks-setup*

Example with lazy.nvim: >
  {
    "emmanuelchucks/patchmarks.nvim",
    config = function()
      require("patchmarks").setup({
        preview = {
          trigger = "manual",
        },
      })
    end,
  }
<
Requirements:

- Neovim >= 0.10
- Git on $PATH
- MIT licensed, see LICENSE

3. Commands                                                 *patchmarks-cmds*

                                                       *:PatchmarksStart*
:PatchmarksStart
  Start or resume a Patchmarks session for the current Git worktree.

                                                       *:PatchmarksFiles*
:PatchmarksFiles
  Open a native quickfix list for the active session's changed files.

                                                     *:PatchmarksRefresh*
:PatchmarksRefresh
  Explicitly refresh changed-file metadata without discarding annotations.

                                                         *:PatchmarksNew*
:PatchmarksNew
  Start a fresh round and discard existing annotations.

                                                      *:PatchmarksExport*
:PatchmarksExport
  Export annotations to registers and clipboard when available.

                                                        *:PatchmarksStop*
:PatchmarksStop
  Stop Patchmarks UI state but keep the session persisted.

                                                     *:PatchmarksDiscard*
:PatchmarksDiscard
  Delete the persisted session and clear in-memory state.

4. Keymaps                                                  *patchmarks-maps*

These mappings are buffer-local to normal file buffers that belong to the
active Patchmarks session.

`<localleader>a`
  Add an annotation on the current line or on the current visual line range.

`<localleader>e`
  Edit the annotation under the cursor.

`<localleader>d`
  Delete the annotation under the cursor.

`<localleader>p`
  Preview the annotation under the cursor.

`<localleader>x`
  Export the current review.

`<localleader>r`
  Refresh the session.

`<localleader>R`
  Start a new round.

`[a`
  Jump to the previous annotation in the current file.

`]a`
  Jump to the next annotation in the current file.

==============================================================================
5. Workflow                                              *patchmarks-workflow*

1. Open a changed source file normally.
2. Run |:PatchmarksStart| inside a Git worktree.
3. Add annotations with `<localleader>a`.
4. Preview or edit them as needed.
5. Optionally run |:PatchmarksFiles| for a native quickfix changed-file list.
6. Run |:PatchmarksExport| to copy a compact review block.
7. If the last review was exported and Git has changed, |:PatchmarksStart|
   starts a fresh round automatically.
8. Run |:PatchmarksNew| when you want to force a fresh round.

Patchmarks does not make source buffers read-only. Session files remain normal
editable file buffers, so existing tools like gitsigns.nvim, mini.diff, or
Fugitive can keep handling hunk navigation and staging.

==============================================================================
6. Configuration                                          *patchmarks-config*

Patchmarks exposes a small setup surface: >
  require("patchmarks").setup({
    preview = {
      trigger = "manual", -- "manual" | "cursorhold"
      width = 72,
      height = 6,
    },
  })
<
Defaults:

- `preview.trigger = "manual"`
- `preview.width = 72`
- `preview.height = 6`

==============================================================================
7. Notes                                                    *patchmarks-notes*

Session state is persisted under: >
  .git/patchmarks/current.json
<
If the session was already exported and Git has changed since that export,
|:PatchmarksStart| starts a fresh round instead of restoring old annotations.

Supported in v1:

- modified, added, renamed, copied, and untracked files
- single-line and line-range annotations
- ordinary editable source buffers

Not supported in v1:

- deleted tracked files
- overlapping annotations
- automatic carry-forward across rounds
- agent-specific integration

Patchmarks is inactive in non-file buffers such as quickfix, help, terminal,
prompt, nofile, acwrite, and plugin-owned buffers.

Annotation editor behavior:

- `:w` saves and keeps the editor open
- `:wq` and `ZZ` save and close
- `:q!` and `ZQ` discard

Empty-body behavior:

- new annotation: empty save/quit is a no-op
- existing annotation: empty `:q` keeps the original annotation
- existing annotation: empty `:w` or `:wq` asks whether to delete it
